# 回合制谜题关卡编辑与回放工具

一款纯前端浏览器应用，用于设计推箱子、开关谜题等回合制关卡。支持网格地图编辑、机关规则配置、解法录制、撤销重做、导入导出及本地持久化。

## 快速开始

```bash
npm install
npm run dev
```

然后浏览器访问 `http://localhost:5173`。

构建生产版本：`npm run build && npm run preview`。

## 界面概览

| 区域 | 模块 | 功能 |
|------|------|------|
| 顶部 | 工具栏 | 关卡名编辑、地图尺寸调整、撤销/重做、新建/样例/导入/导出/保存/校验 |
| 左侧 | 工具箱 | 8 种图块（擦除、墙、起点、目标、箱子、机关、门、地板），数字键 1-8 切换 |
| 中央 | 网格画布 | 点击/拖拽绘制图块，滚轮缩放 0.5x-2.0x |
| 右侧 | 规则面板（可折叠） | 胜利条件、箱子规则、机关-门关联 |
| 底部 | 控制栏 | 方向键十字、步骤时间轴、录制/停止、清除、重置 |
| 右上角 | Toast 通知 | 操作反馈、校验错误、规则变更提醒 |

快捷键：
- `1-8`：切换图块工具
- `Ctrl+Z`：撤销
- `Ctrl+Y`：重做
- `WASD` / `方向键`：录制模式下移动玩家

## 场景一：创建并导出有效关卡（推荐流程）

1. 启动后默认进入空白关卡。点击工具栏 **样例 ▼** → **新手教程** 加载预置的 7×7 推箱子关卡，或从空白开始创建：
   - 工具 2 **墙（🧱）**：在外围围一圈墙
   - 工具 8 **地板（🟫）**：在墙内铺设可通行区域
   - 工具 3 **起点（🚶）**：放置一个玩家起点
   - 工具 4 **目标（🎯）**：放置两个目标点
   - 工具 5 **箱子（📦）**：放置两个箱子
2. 点击右上角 **⚙** 打开规则面板，**胜利条件** 选择 **所有箱子到目标**（默认）。
3. 点击工具栏 **✓ 校验**。若通过，校验按钮图标变绿；不通过则右上角红色 Toast 给出具体错误原因（不会修改你正在编辑的地图）。
4. 点击底部 **⏺ 录制**（若校验未通过会阻止并弹出原因）。
5. 用 **WASD / 方向键** 或底部十字按钮推动箱子到目标。
   - 若**撞墙**、**走出边界**或**推到被阻挡的箱子**：右下角红色 Toast 提示原因，当前步骤不会被记录。
6. 两箱均到达目标后，录制按钮自动变停止并弹出胜利 Toast。
7. 点击工具栏 **💾 保存草稿** 存入 localStorage（操作后 500ms 也会自动保存）。
8. 点击工具栏 **📥 导出**，下载 `<关卡名>.json`。再次导出同一张关卡，JSON 内容除 `updatedAt` 时间戳外完全一致。

## 场景二：构造无效关卡并观察校验反馈

按下列方式设置地图，**每次点击校验**，右上角红色 Toast 会显示第一个具体的错误原因：

| 错误场景 | 操作 | 预期错误码 / 消息 |
|----------|------|-------------------|
| **没有目标（E001）** | 新建关卡不放置任何 🎯，直接点击校验 | `未设置目标点。请至少放置一个目标` |
| **两个玩家（E002）** | 放置两个 🚶 起点 | `存在 2 个起点，只能有一个` |
| **没有起点（E002）** | 用擦除工具抹掉起点 | `缺少玩家起点。请放置一个起点` |
| **元素越界（E003）** | 手工编辑导出 JSON 将 `playerStart.x` 改为 `-1` 再导入 | `起点位置 (-1,0) 超出边界` |
| **玩家/箱子被完全包围（E004）** | 用四面墙把 🚶 起点围死再校验 | `玩家起点被墙或门完全包围，无法移动` |
| **机关引用不存在（E006）** | 规则面板点「添加关联」再删除该机关，或手工编辑 JSON 写出不存在的 switchId 再导入 | `机关规则引用了不存在的机关ID` |

### 校验不污染当前关卡

所有校验都在**深拷贝副本**上执行（见 `validator.ts` `validateLevel`），UI 只通过 Toast 显示错误，绝不修改 `present.tiles`、`boxes` 或撤销栈。

## 场景三：撤销 / 重做（Undo / Redo）

1. 打开 **新手教程** 样例。
2. 用工具 2（墙）在地图中央任意位置画几块墙。每一笔单击或拖拽的连续绘制，都会推入一条历史快照。
3. 按 **Ctrl+Z** 或工具栏 **↶**：墙消失，`present` 状态回滚到 `past` 最后一个快照。
4. 再按 **Ctrl+Z** 多次，直到撤销按钮变灰（`past.length === 0`）。
5. 按 **Ctrl+Y** 或工具栏 **↷**：未来快照依次重放，直到 `future.length === 0`。
6. 在已经撤销后重新绘制会**清空** `future` 分支，不可再重做之前的分支。
7. 刷新页面：
   - `localStorage['puzzle-editor:v1:state']` 会完整恢复 `past / present / future / lastValidation`；
   - 撤销栈、关卡名、校验结果、导出 JSON 与刷新前完全一致。

撤销/重做对下列操作生效（所有会推入撤销栈的动作）：
- 设置图块（含拖拽批量）
- 调整地图尺寸
- 更改胜利条件
- 添加 / 删除机关-门关联
- 清除解法步骤
- 开始录制（清空 moveLog 的动作）
- 录制期间每一次成功移动

撞墙、出界、被门阻挡等**非法移动**不会推入撤销栈，也不会修改 `moveLog`，因此不会污染历史。

## 场景四：规则变更后旧步骤失效

1. 打开 **新手教程** 样例，**校验 → 录制 → 至少推 5 步**。底部步骤栏显示 `↑1 →2 ↓3 …` 高亮序列。
2. 点击右侧 **⚙** 打开规则面板，把胜利条件从 **所有箱子到目标** 改为 **到达目标位置**。
3. 结果：
   - 右上角琥珀色 Toast：`规则已变更，已有解法步骤标记为失效，请重新录制`。
   - 底部步骤栏所有 chip 变为灰色删除线 + 红色 **失效** 标签（仍保留供参考）。
   - 此时尝试按方向键会被拒绝并弹出红色 Toast：`解法步骤已失效，请清除后重新录制`。
4. 点击底部 **清除**：
   - 旧步骤被清空（该操作可撤销）；
   - `moveLogInvalidated` 重置为 `false`，可再次点击 **录制** 重新录制。
5. 规则变更（胜利条件、机关-门关联）会写入撤销栈，所以可以 Ctrl+Z 撤销规则改动；撤销后 `moveLogInvalidated` 会被恢复为当时的值，旧步骤也随之**恢复为有效**。

## 场景五：导入格式错误（不污染当前关卡）

1. 打开记事本创建一个空文件，内容写入：
   ```
   { this is not valid JSON !!!
   ```
   另存为 `bad.json`。
2. 回到编辑器，先加载一个有内容的样例（例如 **新手教程**）。
3. 点击工具栏 **⇪ 导入**，选择 `bad.json`。
4. 结果：
   - 右上角红色 Toast：`导入失败：JSON 解析失败：格式不合法`；
   - 当前样例关卡完全不变（地图、步骤、撤销栈都未被改动）。
5. 再创建一个结构合法但缺字段的 `malformed.json`：
   ```json
   { "version": "1.0.0", "name": "缺字段", "width": 3, "height": 3 }
   ```
6. 再次导入。Toast 会逐条列出缺失字段：`缺少 tiles 字段或不是数组` 等。当前关卡依旧不变。
7. 最后导入一个**合法**导出 JSON：
   - 先按步骤一导出一份 `新手教程.json` 备份；
   - 新建一个空关卡；
   - 导入备份，当前关卡被正确替换，撤销栈将「导入前的空关卡」保留为可撤销节点。

## 持久化与刷新一致性

刷新 / 重开页面后以下内容会完全一致：

| 项目 | 存储键 | 说明 |
|------|--------|------|
| `past / present / future` | `puzzle-editor:v1:state` | 完整撤销栈 + 当前关卡 + 重做栈；每条历史记录同时保存对应时刻的校验结果 |
| `lastValidation` | 同上 | 上次校验结果（有效/无效/错误列表） |
| `moveLog`、`moveLogInvalidated` | 同上（内嵌于 `present`） | 解法步骤及失效标记 |
| 最后自动保存时间 | `puzzle-editor:v1:last-saved` | `Date.now()` 毫秒 |

每次 `present` 引用变更 **或** `lastValidation` 校验结果变化后 500ms 内自动写入 localStorage。导出 JSON 可用于跨设备分享、版本管理、或作为单元测试 fixture。

## 项目结构

```
src/
├── types/index.ts           // 类型定义、TileType、Direction、WinCondition、LevelData
├── utils/
│   ├── mapOps.ts            // 创建/克隆/调整地图，查找位置，重建派生数据
│   ├── simulation.ts        // 移动模拟、推箱子、门状态、胜利判定、步骤重放
│   ├── validator.ts         // E001-E006 校验引擎（纯函数，不修改原对象）
│   └── serializer.ts        // 导入/导出、3 个样例关卡、旧格式迁移
├── store/useEditorStore.ts  // Zustand store：撤销栈、持久化中间件、所有动作
├── components/
│   ├── Toolbar.tsx          // 顶部工具栏
│   ├── Toolbox.tsx          // 左侧工具箱
│   ├── GridEditor.tsx       // 中央网格画布（绘制 + 模拟覆盖层）
│   ├── RulesPanel.tsx       // 右侧规则配置（滑入抽屉）
│   ├── ControlBar.tsx       // 底部方向键 + 步骤时间轴 + 录制控制
│   └── Toast.tsx            // 全局通知
├── pages/Home.tsx           // 主工作台布局 + 全局快捷键
└── index.css                // 玻璃拟态组件类（glass-panel、btn-*、chip 等）
```

## 校验错误码一览

| 码 | 说明 |
|----|------|
| E001 | 缺少目标点 |
| E002 | 起点缺失或多于一个 |
| E003 | 元素坐标超出边界 |
| E004 | 玩家或箱子被完全包围无法移动 |
| W001 | 箱子数与目标数不一致（警告，不阻塞） |
| E006 | 机关规则引用不存在的 switchId |
| W002 | 机关未配置关联规则（警告，不阻塞） |
| JSON 解析 / 结构错误系列 | 导入阶段产生，见 importFromJSON |
